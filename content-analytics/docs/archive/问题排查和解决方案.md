# 后端连接问题排查

## 问题描述
前端显示错误："无法连接服务器，请确认后端服务已启动"

## 排查结果

### ✅ 后端服务状态
- **后端服务正在运行** ✅
- **端口**: 8088
- **健康检查**: http://127.0.0.1:8088/health 返回 200 OK
- **进程ID**: 1724

### ⚠️ 前端代理问题
前端配置了代理将 `/api` 请求代理到 `http://localhost:8088`，但浏览器无法连接。

## 解决方案

### 方案1：重启前端服务（推荐）

前端开发服务器需要重启才能使用代理配置。请执行以下步骤：

1. **停止前端服务**
   - 在运行 `npm run dev` 的终端窗口中按 `Ctrl+C`

2. **重新启动前端服务**
   ```bash
   cd content-analytics/frontend
   npm run dev
   ```

3. **等待服务启动完成**
   - 看到 "Local: http://localhost:5173/" 表示启动成功

4. **刷新浏览器页面**
   - 按 `F5` 或 `Ctrl+R` 刷新页面

### 方案2：检查后端服务日志

如果重启前端后仍有问题，检查后端服务是否有错误：

1. 查看运行后端服务的终端窗口
2. 检查是否有错误日志
3. 确保数据库连接正常

### 方案3：直接访问后端API测试

在浏览器中直接访问后端API测试：
- http://localhost:8088/health
- http://localhost:8088/api/docs

如果这些URL可以访问，说明后端正常，问题在前端代理配置。

## 快速检查清单

- [ ] 后端服务运行在 8088 端口
- [ ] 前端服务运行在 5173 端口  
- [ ] 前端服务已重启（应用代理配置）
- [ ] 浏览器已刷新页面
- [ ] 没有防火墙阻止连接

## 如果问题仍然存在

1. 检查 Windows 防火墙是否阻止了连接
2. 尝试清除浏览器缓存
3. 检查是否有其他程序占用了 8088 或 5173 端口
4. 查看浏览器控制台（F12）的错误信息
